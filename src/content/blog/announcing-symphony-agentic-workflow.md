## The zero-code delivery path

Symphony aligns four pillars into one continuous loop:

1. **Linear**: Product work begins in Linear issues. Symphony reads context, updates status, and keeps a single source of truth for progress.
2. **Symphony + Codex**: AI copilots translate requirements into implementation plans, modify code, and run validations autonomously.
3. **GitHub**: Changes land in clean branches and pull requests with status checks, diffs, and review hooks intact.
4. **Vercel**: Deployments ship to Vercel automatically, giving you preview URLs and production pushes without manual steps.

## Why it matters

- **Hands-free execution**: From intake to deploy, Symphony keeps the loop moving—no juggling tabs or glue scripts.
- **Traceable by default**: Issue updates, commits, PRs, and deploys stay linked, so audit trails and reviews are effortless.
- **Faster iteration**: Codex accelerates coding and testing, while Vercel previews make feedback instantaneous.
- **Team-friendly**: Product managers stay in Linear, engineers review in GitHub, and releases go live on Vercel—everyone sees the same truth.

## How it works day-to-day

- Capture a feature or fix in Linear.
- Symphony spins up Codex to plan tasks, edit code, and run checks locally.
- The bot opens a GitHub branch and pull request with context-rich summaries and linked issue updates.
- Successful checks trigger Vercel previews, and merging to main ships to production.

## Impact you can feel

This agentic workflow removes coordination tax. Product teams express intent in Linear; Symphony and Codex handle execution; GitHub guards quality; Vercel delivers. The result is a zero-code delivery system where ideas become running software with speed, safety, and almost no manual effort.
